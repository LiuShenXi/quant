# 10 · Codex v1.1 修改说明

## 1. 总体判断

原架构方向不需要推翻。它适合个人中低频量化系统:模块化单体、策略契约、事件驱动回测、统一数据层、OMS/风控/对账优先。这些核心选择保留。

本次修订只做"落地前补硬":把容易在代码实现阶段造成歧义的边界、字段和验收标准写清楚。

## 2. 关键修改

1. 契约包统一为 `core.contract`,策略只允许依赖这个包。
2. `Context` 不再暴露 `mode`,避免策略按 backtest/paper/live 写分支。
3. `Order/Trade/Position/Account` 增加策略实例、账户、券商单号、拒单原因、更新时间、剩余数量等实盘必要字段。
4. 数据层增加时间和口径约定:默认 `Asia/Shanghai`, `Bar.dt` 表示 bar 结束时间,`history()` 必须物理截止到 `end`。
5. 数据表增加 `data_status/source/updated_at`,把停牌、无成交、数据缺失区分开。
6. 回测产物增加 `events.jsonl`,尽量与实盘事件流水同结构。
7. OMS 明确负责把券商原始状态规范化为契约事件,策略不能接触券商原始对象。
8. 风控拒单必须结构化返回 `reject_reason` 和 `risk_rule_id`。
9. 路线图把 M0/M1 调整为最小可信闭环,把 QMT/真金实盘后置。

## 3. 没有改的东西

1. 不引入微服务、消息队列、容器编排。
2. 不把研究层工程化到过重。
3. 不提前做 Web 管理台、tick 高频、多账户、期权和复杂机器学习管线。
4. 不改变 A股/QMT 优先、其他市场通过 Gateway 替换的方向。

## 4. 下一步建议

如果开始写代码,第一步只做 M0/M1:

1. 建 `quant/` 真实项目和 git repo。
2. 实现 `core.contract` 并写契约测试。
3. 做一个固定小数据集和 `DataService.history()`。
4. 跑通 DualMA 事件回测。
5. 建立黄金回归测试。

这一步完成前,不要接 QMT,也不要上真钱。
