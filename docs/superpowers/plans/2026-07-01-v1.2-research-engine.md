# v1.2 Research Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first v1.2 research-engine vertical slice from the architecture package while preserving current A-share ETF daily backtest, paper-prep, risk, event, and report behavior.

**Architecture:** This is an incremental modular-monolith extension, not a rewrite. The platform gains reusable research/backtest capabilities for configurable calendars, dataset manifests, multi-frequency bars, no-lookahead clocks, target weights, quote-currency accounting, bps costs, portfolio risk overlay, append-only events, and generic reports. Strategies must continue to depend only on `quant.core.contract` and whitelisted third-party libraries.

**Tech Stack:** Python, Pydantic, pandas, pytest, existing `src/quant` modular monolith.

## Global Constraints

- Current worktree: `C:\WORK-SPACE\quant\.worktrees\v1.2-research-engine`.
- Branch: `codex/v1.2-research-engine`.
- Architecture package: `版本迭代架构包/v1.2-research-engine/`.
- First round only allows research/backtest tooling; no exchange, broker, live gateway, QMT, ccxt, real credentials, live overlay, paper promotion, M3b signoff, or real-money execution.
- Keep existing modular monolith boundaries: `core`, `data`, `backtest`, `risk`, `live`, `strategies`.
- Strategies may import only `quant.core.contract` and whitelisted third-party libraries. They must not import `data`, `backtest`, `live`, `risk`, or `ops`.
- Platform code must not import or reference concrete strategy IDs such as `crypto_trend_breadth_top2_v1`.
- Engine code must not hard-code BTC, ETH, SOL, 4h, daily, EMA, 60/40, 20% stop, 5-day cooldown, or 35% red line.
- Keep legacy `freq: "1d"` strategy configs compatible by mapping to `frequencies.primary = "1d"`.
- `Bar.dt` continues to mean bar end time.
- `ctx.history(symbol, n, freq)` may return only closed bars with `dt <= ctx.now`.
- Any target/order must not fill on the same bar that produced the signal.
- Calendar/session behavior comes from config or metadata, not A-share hard-coded assumptions.
- Account currency, settlement, quote currency, quantity step, lot size, and fractional rules come from config or instrument metadata.
- Monetary risk limits must declare unit: `quote_currency`, explicit `currency`, or `equity_pct`.
- All order, trade, rejection, risk action, cash transition, engine state, and target-intent state changes must be append-only event records.
- `report.json` must contain `not_trading_permission: true`; `report.md` must include a research-only disclaimer.
- Every task must follow TDD: write failing test, run and confirm expected failure, implement minimally, rerun targeted tests, then run the required legacy tests for that slice.
- The baseline suite passed before this plan: `pytest -q` -> `222 passed`.
- Required legacy regression command after relevant slices: `pytest tests/test_backtest_engine.py tests/test_data_service.py tests/test_risk_pipeline.py tests/test_portfolio_costs.py tests/test_import_boundaries.py -q`.

## Director Dispatch Rules

- Dispatch implementation workers serially by task number because tasks share core files and later tasks consume earlier interfaces.
- Use explorer agents in parallel only for read-only inspection and final risk-focused review.
- After each worker returns, generate a review package from the recorded task base SHA to task head SHA.
- Dispatch a reviewer for that task with the task brief, worker report, and review package.
- Do not mark a task complete until reviewer gives both spec compliance and code quality approval, or all Critical/Important findings are fixed and re-reviewed.
- Record durable progress in `.superpowers/sdd/progress.md`.

- If a reviewer finds a plan-mandated issue, present the conflict to the human operator instead of overriding the plan.
- Final acceptance requires:
  - all ten task reviews clean;

  - no paper/live/QMT/exchange/broker/real-money code path introduced;
